# SportPit — Протокол Егорова

Веб-приложение для планирования питания по протоколу Егорова (16:8 + циклирование углеводов/жиров).

## Стек

- React + Vite + TypeScript
- Tailwind CSS v4
- Vercel serverless functions (`@vercel/node`)
- Neon Postgres (`@neondatabase/serverless`)
- JWT-аутентификация (`jsonwebtoken`, `bcryptjs`)

## Переменные окружения (Vercel)

```
DATABASE_URL=postgresql://...  # строка подключения к Neon
JWT_SECRET=...                 # случайная строка для подписи токенов
```

## Схема базы данных

Запустить `schema.sql` в консоли Neon перед первым деплоем.

## Разработка

```bash
cd sportpit
npm install
npm run dev
```

## Деплой

```bash
cd sportpit
vercel --prod
```

## Баланс белка 60/40

Приложение учитывает соотношение животного и растительного белка (~60/40). Для этого поддерживаются:

- **Животные источники:** курица, индейка, говядина, рыба, яйца, творог, сыр, протеин.
- **Растительные источники:** тофу, соя, нут, фасоль, чечевица, грибы, орехи и семечки.

Каждый день в карточке плана отображается, сколько белка пришлось от животных и растительных продуктов.
